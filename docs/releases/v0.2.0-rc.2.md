# v0.2.0-rc.2 Compose-first operations

This release candidate makes Docker Compose project directories the recommended authority for agent-managed homelab services.

## Included

- a machine-readable catalog operating model
- the `agent-homelab recipe policy` command
- Compose-first setup, service lifecycle, and agent-skill guidance
- an unchanged Portainer CE recipe for optional visibility and manual diagnostics

## Operating recommendation

Keep each service in an operator-owned Docker Compose project directory. Store the Compose definition, environment references, image pins, state mapping, and recovery notes there.

Use that directory for validation, startup, upgrades, backup mapping, and recovery. Do not make Portainer's database the only definition of an agent-managed stack.

## Compatibility

This release keeps the Python, Traefik, Authelia, Docker Compose, BorgBackup, and BusyBox compatibility ranges from `v0.2.0-rc.1`.
